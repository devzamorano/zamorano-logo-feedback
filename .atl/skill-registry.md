## react-19
Trigger: writing React components.
- No manual memoization — React Compiler handles it. Never `useMemo`/`useCallback`.
- Named imports only: `import { useState } from "react"` — never `import React from "react"`.
- Server Components are the default; add `"use client"` only for useState/useEffect/refs, event handlers, or browser APIs.
- Use `use()` for reading promises/conditional context instead of `useContext` when conditional.

## tailwind-4
Trigger: styling with Tailwind.
- Never use `var(--x)` or hex colors inside `className`. Use Tailwind semantic classes (`bg-primary`, `text-slate-400`).
- Use `cn()` (clsx + tailwind-merge) only for conditional/mergeable classes — not for static class strings.
- Dynamic numeric values (e.g. computed widths) go in `style={{}}`, not className.

## typescript
Trigger: writing TypeScript.
- Const-object + derived union type pattern instead of raw string unions: `const STATUS = {...} as const; type Status = (typeof STATUS)[keyof typeof STATUS]`.
- Flat interfaces — nested objects get their own named interface, not inlined.
- Never use `any`; use `unknown` + narrowing or generics.

## zod-4
Trigger: schema validation.
- Zod 4 top-level validators: `z.email()`, `z.uuid()`, `z.url()` — not `.string().email()` etc. (breaking change from v3).
- `z.string().min(1)` replaces `.nonempty()`.
- Error messages passed as `{ error: "..." }`, not `required_error`/`invalid_type_error`.
